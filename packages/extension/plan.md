# Analytics Dashboard Plan

## Концепция

Отдельная полноэкранная страница `dashboard.html`, открывается из попапа по кнопке "📊 Analytics" или из иконки в навигации. Даёт полный обзор продуктивности с графиками и детальной статистикой.

## Структура

```
src/
  dashboard/
    dashboard.html
    dashboard.ts
    dashboard.css
    components/
      charts.ts      # Графики (SVG/Canvas)
      stats.ts       # Карточки статистики
      history.ts     # История задач
```

## Данные

Используем существующие структуры:
- `PomodoroRecord[]` — история помодоро (taskId, date, duration)
- `DailyStats[]` — агрегированная статистика по дням
- `Task[]` — задачи с totalWorkTime, pomodoroCount, completedAt

## Этапы реализации

### Phase 1: Базовая страница
- [ ] Создать `dashboard.html` с базовой структурой
- [ ] Добавить в `manifest.json`
- [ ] Кнопка открытия в попапе (иконка 📊 в навигации)
- [ ] Базовые стили (тёмная тема как в попапе)

### Phase 2: Карточки статистики
- [ ] Общее количество помодоро за период
- [ ] Общее время работы
- [ ] Количество активных дней
- [ ] Текущий streak

### Phase 3: Графики
- [ ] Bar chart — помодоро по дням (SVG)
- [ ] Pie chart — распределение по задачам (SVG)
- [ ] Переключатель периода (Today/Week/Month)

### Phase 4: История задач
- [ ] Таблица завершённых задач
- [ ] Сортировка по дате
- [ ] Показ времени и количества помодоро

### Phase 5: Улучшения
- [ ] Heatmap календарь (как на GitHub)
- [ ] Экспорт данных (JSON/CSV)
- [ ] Цели (daily/weekly targets)

## Технические детали

**Графики:** Чистый SVG без библиотек (легковесно, полный контроль)

**Открытие дашборда:**
```typescript
chrome.tabs.create({ url: chrome.runtime.getURL('dashboard/dashboard.html') });
```

**Стек:** TypeScript + Tailwind (как в попапе)

## Приоритет

1. Phase 1-2 — MVP с базовой статистикой
2. Phase 3 — графики (главная ценность)
3. Phase 4-5 — по желанию
